GPT BERT and BART

This note covers the evolution of three foundational Transformer architectures: GPT, BERT, and BART. We will look at how they differ in their attention mechanisms, training objectives, and whether they use encoder, decoder, or full sequence to sequence structures. Let's break down the details of pre training, fine tuning, and why each architecture is suited for specific tasks.

Timeline

GPT

Generative Pre trained Transformer (see paper: https://cdn.openai.com/research-covers/language-unsupervised/language_understanding_paper.pdf)

GPT is a decoder only transformer that is autoregressive. It predicts the next token in a sequence given all previous tokens. The key idea is to leverage unsupervised pre training on large corpora and then perform supervised fine tuning (SFT) on specific tasks such as question answering.

GPT uses masked self attention to predict tokens autoregressively. Each token can only attend to previous tokens. This allows the model to build context from what has already been generated while preventing future token leakage.

Stages

Pre training (Unsupervised):

  1. Input tokens → embeddings + positional encodings
  2. Multiple decoder blocks (masked self attention + feedforward networks)
  3. Linear projection → softmax → next token probabilities
  4. Teacher forcing: at each timestep, the correct token is provided as input for predicting the next token
  5. Loss: cross entropy between predicted probabilities and true tokens

Supervised Fine Tuning (SFT):

  1. Input sequence: task specific text (e.g., question)
  2. Target: desired output (e.g., answer)
  3. The same embedding and decoder weights are used
  4. Each token in the input sequence is converted to embeddings
  5. Teacher forcing continues: at each step, the correct target token is provided to predict the next
  6. Fine tuning updates weights to align pre training knowledge with the task

Decoder Architecture

Since it is built on the same decoder as the transformer, it uses the Masked Multi Head Self Attention architecture, here is a quick recap of the Decoder.

The decoder only architecture consists of stacked blocks:

  1. Masked Multi Head Self Attention
  2. Residual connection + LayerNorm
  3. Feedforward network
  4. Residual connection + LayerNorm
  5. Linear projection + Softmax

Pre training and Fine Tuning

Pre training

Any GPT is first trained on a huge corpus of text containing sentences. Its training objective is to predict the next token. Pre training is done to build up context about the language for the transformer. By training on a huge corpus of data to predict the next token, the model understands when to generate tokens given a piece of context.

Pre training Objective:

The objective is to maximize the following likelihood: $$L_1(U) = \sum_{i} \log P(u_i \mid u_{i-k}, \dots, u_{i-1}; \Theta)$$ where $k$ is the size of the context window, and the conditional probability $P$ is modeled using a neural network with parameters $\Theta$. These parameters are trained using stochastic gradient descent.

The training objective is to predict a token given a set of tokens (predicting a sentence/token given a certain context).

For Example: Given the sentence "The capital of France is", the model would predict the token "Paris".

Supervised Fine Tuning

After building an understanding of the language using pre training, we fine tune the model so that it can perform specific tasks. The most commonly used task is Q and A where you prompt an LLM with a question and it answers.

Fine tuning is done by training the already trained weights/embeddings on specific use case data. In the case of Q and A, you give it Question Answer pairs so that it can learn to answer the questions from the facts and language built up in the pre training stage.

Suppose we have a labeled dataset $C$, where each instance consists of an input sequence of tokens $x^1, \dots, x^m$ and a target label $y$. The learning objective is to maximize: $$L_2(C) = \sum_{(x, y)} \log P(y \mid x^1, \dots, x^m; \Theta)$$ Given an input sequence, the model should give high probability to the correct output $y$.

Auxiliary Loss

During fine tuning, we use an auxiliary loss function along with the task specific loss function. This is so that the model retains what it has learned during the pre training stage and to avoid overfitting.

An auxiliary loss function ensures that the model does not forget the language model it has built in the pre training stage and instead uses this to learn the task it is being trained for.

So during fine tuning we use a task specific loss: $$\mathcal{L}_{\text{task}} = - \sum_{t=1}^{T} \log p_\theta(y_t \mid y_{<t}, x)$$ And an auxiliary loss (which is the same as the loss we used in pre training): $$\mathcal{L}_{\text{LM}} = - \sum_{t=1}^{T} \log p_\theta(x_t \mid x_{<t})$$

The total loss that we train for is: $$\boxed{\mathcal{L}_{\text{total}} = \mathcal{L}_{\text{task}} + \lambda \, \mathcal{L}_{\text{LM}}}$$ where $\lambda$ is the balance weight that controls how much of the auxiliary loss we want the model to be influenced by.

Token Prediction

  • Context vector from final decoder block: $$h \in \mathbb{R}^{T \times d_{\text{model}}}$$

  • Linear projection: $$\text{Logits} = h W^T + b$$

  • Softmax → probabilities over vocabulary: $$P(y_t = w_i \mid \text{context}) = \frac{\exp(\text{logit}_i)}{\sum_{j=1}^{V} \exp(\text{logit}_j)}$$

  • The most probable token is selected (greedy, sampling, or beam search)

Why This Works

  1. Pre training:
    • Learns general language patterns, syntax, semantics, and world knowledge
    • Each token attends to relevant context → rich contextual embeddings
  2. Fine tuning:
    • Updates weights to adapt general knowledge to specific tasks
    • Input/output sequences mapped to pre trained embeddings, allowing transfer learning
  3. Autoregression + Teacher Forcing:
    • Gradually builds sequence understanding
    • Prevents the model from "cheating" by seeing future tokens
    • Enables robust next token prediction and Q/A generation

Intuitive Understanding:

Pre training is like training a child on the vocabulary of a language where the model tries to replicate how certain words are used in certain contexts.

Fine tuning is like teaching the child to form full fledged sentences and answer questions based on the language learned.

BERT

Bidirectional Encoder Representations from Transformers uses the transformer architecture specifically to understand language and build up an understanding of the input data.

Since it is an encoder only architecture, its purpose is to specifically build up an understanding of the input corpus. Traditionally, transformers process text sequentially from left to right.

However, this often limits the model's ability to achieve true language understanding. So, BERT uses a bidirectional approach considering both the left and right context of words in a sentence. Instead of analyzing the text sequentially, BERT looks at all the words in a sentence simultaneously. It uses a bidirectional self attention mechanism.

To make BERT look at tokens in both directions in a sentence, we remove the traditional no look mask training objective and introduce 2 pre training objectives:

Masked Language Modeling

Randomly mask some fraction of input tokens with a special [MASK] token. The token chosen can be either replaced with [MASK], replaced with a random token, or kept unchanged. At any point, 15% of the tokens are selected. Out of these, 80% are replaced with [MASK], 10% with a random token, and 10% are kept unchanged. The training objective is to predict the correct original token for these selected positions using both left and right context.

$$P(x_{\text{masked}} \mid x_{\text{left}}, x_{\text{right}})$$

Next Sentence Prediction

MLM gives token level bidirectionality, but BERT also needs sentence level coherence for tasks like QA and entailment.

BERT is given sentence pairs. 50% of the time the actual sentence B that follows A is given, and the other 50% of the time it is a random sentence from the corpus. BERT must predict a binary label saying whether B is the correct sentence that follows A.

MLM helps the model understand tokens on the basis of both past and future (left and right) context, whereas NSP uses that understanding of tokens and applies it to understanding relations between entire sentences.

The final pre training loss is the sum of MLM and NSP loss.

$$\mathcal{L}_{\text{BERT}} = \mathcal{L}_{\text{MLM}} + \mathcal{L}_{\text{NSP}}$$

BART

Bidirectional Autoregressive Transformer is an autoencoder which takes in a corrupted version of a text sequence into its encoder, maps it to a latent space, and then uses the decoder to generate the corrected version of the text.

The encoder takes in a sequence of tokens which are corrupted. Many types of corruptions are applied like:

  • Token masking (like BERT MLM)
  • Token deletion
  • Token permutation / shuffling
  • Text infilling (span masking)
  • Sentence permutation

The encoder then produces contextual embeddings for each corrupted token which contain the semantic meaning of tokens and the relationships between tokens. It uses self attention.

The decoder is then trained to generate the correct text sequence using teacher forcing (by shifting tokens one step to the right). It uses masked self attention and cross attention to decode the outputs and generate a token.

This architecture is very similar to the original Transformer architecture, but instead of being trained for machine translation, it is pre trained to take in a corrupted/noisy text sequence and reconstruct the original text.

For example:

Corrupted input: "The [MASK] brown [MASK] jumps over the lazy"

Original text to be trained on: "The quick brown fox jumps over the lazy dog"

Why this objective works:

  • Bidirectional encoder learns contextual representations even with missing/shuffled tokens
  • Decoder learns to generate coherent text based on the encoder context.

This forces the model to understand structure and semantics even better than just generating the next token.

BART can be seen as kind of a mixture of BERT and GPT wherein we make use of the bidirectional encoder and generative decoder to build BART.

Overall, GPT's autoregressive nature makes it the go to for generation, BERT's bidirectional context makes it king for comprehension and classification, and BART's denoising seq2seq design strikes a balance for tasks like summarization and translation. Choosing the right model really just comes down to whether you need to understand text, generate text, or do a mix of both.
