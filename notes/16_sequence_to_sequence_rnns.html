Sequence to Sequence RNNs

Table of Contents
Encoder: Decoder: Note: Training:

Sequence 2 Sequence architecture is used in RNNS in tasks such as translating data in one form to another, summarization and so on

It takes an input sequence processes it and generates an output sequence It has mainly 2 components:

  1. Encoder: Processes the data and encodes it into a vector after the processing
  2. Decoder: Takes the encoded information as input and generates an output sequence

Encoder:

It takes in the pre processed data in the form of vectors (after embedding is done). Since Encoders and Decoders are both RNNS essentially, it takes in data one token at a time and builds up context just like an RNN would It consists of hidden states, timestamps and multiple layers to build up the memory/context of what the input is and give a final hidden state.

Decoder:

Decoders take this final hidden state as input applies RNN operations on it and then it applies a linear layer to the output. RNN Step $h_t$ = RNN(embedding(prev_token), h_{t - 1}) Linear Layer: logits = W_out @ $h_t$ + b

Softmax and prediction next_token=argmax(softmax(logits))

This next token is again taken as input for in the next RNN cycle in the decoder

This cycle is repeated till we reach the End of the Sequence token

Note:

  1. Encoder and Decoder might either have the same set of embeddings or a different set depending on what we're using the model for
  2. If its for translation, obviously the encoder will have an English embedding matrix and the decoder will have a french embedding matrix

Training:

The main strategy that is used in training is something called Teacher forcing In this strategy, instead of feeding the prediction it did in each decoder step as the input to the next decoder step, we feed it the actual correct prediction.

So for every decoder step: decoder_input is the true token (the actual correct answer)

This makes training a lot easier because we're essentially telling the model that for 'this context' the answer should be 'this' For example for the context 'how are you?' we're forcing it to learn that the answer is 'im good'

Example:

Suppose the input given is : "hello, how are you?" the expected output is "im good"

After encoding is done and the context is built up, we get a final context vector from the encoder. So, At time step hidden_1=RNN(embedding(BOS),h_enc) (where BOS is the start token of any decoder which stands for beginning of sequence) logits_1=W_out*hidden_1 +b

logits gives us the prediction of the first word ideally, the answer should be 'im' (token id= 101) loss is computed: loss_1=CrossEntropy(logits_1,101)

At time step 2

Here is where teacher forcing comes in Instead of giving logits_1 as the next input (because most of the time it is wrong and so obviously the next prediction would also be wrong), we give it the correct answer that is 'im' as input instead

hidden_2=RNN(embedding("im"),hidden_1) logits_2=W_out * hidden_2 +b

Ideally, the answer that it should give out is 'good' (token id=275)

So the loss is computed: loss_2=CrossEntropy(logits_2,275)

total_loss =loss1 +loss 2

total_loss.backward() is used to call backprop and backpropagation takes place using gradient descent or whatever and the weights are updated to match the correct answers

How the code part would look like: target_seq = ["BOS", "i", "am", "good"] target_y = ["i", "am", "good", "EOS"] decoder_input = embedding("BOS") hidden = encoder_hidden_state # from encoder output loss = 0 for t in range(len(target_y)): output, hidden = decoder_rnn(decoder_input, hidden) # Forward pass logits = output_layer(output)
loss += CrossEntropyLoss(logits, target_y[t])
decoder_input=embedding(target_seq[t +1])

loss.backward() optimizer.step() optimizer.zero_grad()

loss is called so that optimization happens

Gradients flow through:

  1. Decoder output layer (logits->vocab)
  2. Decoder RNN
  3. Decoder input embeddings
  4. Encoder RNN
  5. Encoder embeddings
