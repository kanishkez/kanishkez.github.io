Regularization

Regularization is a set of techniques used to reduce overfitting in machine learning by adding a penalty to the model's loss function, thereby reducing accuracy on training data to increase generalizability so that the model can adapt to a wider range of unseen data.

Overfitting occurs when the model learns to solely minimize loss and fits the training data too closely, capturing noise instead of the underlying distribution.

To add a penalty, we modify the loss function: $$\text{Loss} = \text{Data Loss} + \lambda \cdot \text{Penalty}$$ where $\lambda$ determines how strong we want the penalty to act.

The penalty discourages model complexity so that the network does not overfit. This means that we penalize large weights (which affect predictions too heavily) and overconfident predictions.

For a single weight $w$, gradient descent performs: $$w \leftarrow w - \eta \Big( \underbrace{\frac{\partial \text{Task Loss}}{\partial w}}_{\text{task gradient}} + \underbrace{\lambda \cdot \text{sign}(w)}_{\text{L1 gradient}} \Big)$$ So, the weight changes according to both the task loss and the penalizer.

Total gradient: $$\nabla w = \nabla \text{Task Loss} + \nabla \text{Regularization}$$

Types of Regularization

L2 Regularization

L2 regularization is the most common and simple form of regularization. $$\text{Penalty} = \sum w^2$$

So the loss function becomes: $$\text{Loss} = \text{Data Loss} + \lambda \sum w^2$$

During training, if a weight becomes too large, this penalty pushes the weight down to reduce its size.

If a weight becomes large:

  • $w^2$ becomes extremely large.
  • The loss increases.
  • The optimizer pushes the weight down to minimize the loss.

L2 regularization is widely used in linear regression, logistic regression, and neural networks.

Taking the derivative of the penalty term: $$\frac{d}{dw}(\lambda w^2) = 2\lambda w$$ Thus, the magnitude of the weight determines the strength of the penalty.

If the weight is large (for example, $w = 10$):

  • The gradient is $20\lambda$.
  • The reduction of the weight is highly noticeable.

If the weight is small (for example, $w = 0.002$):

  • The gradient is $0.004\lambda$.
  • The weight difference is negligible.

L1 Regularization

$$\text{Penalty} = \sum |w|$$ $$\text{Loss} = \text{Data Loss} + \lambda \sum |w|$$

Under L1 regularization, small weights are driven to exactly zero, rendering some weights entirely unused.

Derivative of $|w|$: $$\frac{d}{dw}|w| = \begin{cases} +\lambda & w > 0 \\ -\lambda & w < 0 \\ \text{any value in }[-\lambda, \lambda] & w = 0 \end{cases}$$

If a weight is weak: $$\left|\frac{\partial \text{Task Loss}}{\partial w}\right| < \lambda$$ The regularization gradient dominates and the weight is pushed towards zero.

Therefore, L1 regularization causes shrinkage of small weights because their task loss gradient is smaller than the regularization penalty gradient.
