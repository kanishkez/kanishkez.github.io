Non Linearity and Activation Functions

Table of Contents
Why do we need nonlinearity? What are Activation Functions? common activation functions

Let's take the classic XOR example. Here is the XOR table.

$x_1$$x_2$$y$
000
011
101
110
Suppose we want to make a model that predicts this then we would not be able to do so with simple linear regression or classification.

If you plot these points, you’ll see:

  • The “1”s are at $(0,1)$ and $(1,0)$
  • The “0”s are at $(0,0)$ and $(1,1)$

No straight line can separate the 1’s from the 0’s. This means a single perceptron (linear classifier) cannot solve XOR.

so we would need to use non linearity.

Why do we need nonlinearity?

If you just stack multiple linear transformations, you don’t actually get more power.

For example:

$$z^{(1)} = W^{(1)} x + b^{(1)}$$ $$z^{(2)} = W^{(2)} z^{(1)} + b^{(2)}$$

If you combine these: $$z^{(2)} = W^{(2)}(W^{(1)}x + b^{(1)}) + b^{(2)} \ = (W^{(2)}W^{(1)})x + (W^{(2)}b^{(1)} + b^{(2)})$$

This is still just a linear transformation of $x$. So even if you stack 100 linear layers, the whole thing is equivalent to a single linear layer. → The model is still limited to straight line decision boundaries. So what do we do? We need to break this linearity trap. That’s where activation functions come in.

What are Activation Functions?

An activation function is applied after each layer’s weighted sum. It introduces nonlinearity into the network, allowing it to learn complex, curved, and flexible decision boundaries. Basically, non linear activation functions make the learning pattern complex by adding dimensions to the training phase of the network. We are making the network complex so that it can learn complex patterns and features. A good way to visualize this is to think of it this way: Instead of using a line to fit the features and predict something, we're introducing another dimension and seperating the features there.

So instead of: $z = Wx + b$

We do: $a = \phi(Wx + b)$

where $\phi$ is the activation function.

This way, stacking multiple layers actually gives us more expressive power.

common activation functions

the sigmoid function is given by

$$\sigma(x) = \frac{1}{1 + e^{-x}}$$

it squashes any input into the range (0,1). the derivative is $$\sigma’(x) = \sigma(x)(1 - \sigma(x))$$

this means the gradient is highest when the output is around 0.5 and becomes very small when the output is close to 0 or 1. this small gradient is called the vanishing gradient problem. What does sigmoid let through? Lets through: values near zero (centered around 0.5) Blocks: large positive/negative values (saturates at 0 or 1) sigmoid was used in early neural networks and still makes sense when you need probabilities, but it slows down training for deep models.

the tanh function is $$\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$$ it outputs values in the range (-1,1), which makes it zero centered unlike sigmoid. the derivative is

$$\tanh’(x) = 1 - \tanh^2(x)$$

gradients are stronger around zero but still vanish when the input is very large in magnitude. tanh often performs better than sigmoid in practice but still suffers from vanishing gradients in deep networks. What does tanh let through?: Lets through: values near zero (centered at 0) Blocks: large positive/negative values (saturates at -1 or 1)

How does it shape learning? Zero centered output helps with optimization (balances positive/negative signals). Still suffers from vanishing gradients for large $|x|$. Better than sigmoid for hidden layers, but still limited in deep networks.

the relu function is $$\text{ReLU}(x) = \max(0, x)$$ its derivative is

$$\text{ReLU}’(x) = \begin{cases} 1 & x > 0 \ 0 & x \leq 0 \end{cases}$$

relu is the most widely used activation today because it avoids the vanishing gradient problem for positive inputs, making training much faster. but the problem is for negative inputs, the gradient is zero, which means those neurons can “die” and stop learning. this is called the dying relu problem. What does ReLU let through? Lets through: all positive values (unchanged) Blocks: all negative values (outputs zero)

How does it shape learning? Enables sparse activations (many neurons output zero), which can improve efficiency and generalization. Avoids vanishing gradient for $x > 0$ (gradient is 1), but can cause "dead neurons" (if $x < 0$ always, neuron never activates). Default for hidden layers in modern deep networks.

Analogy: Like a one way valve—only lets positive flow through, blocks the rest.

a variation is leaky relu, defined as

$$\text{LeakyReLU}(x) = \begin{cases} x & x > 0 \ \alpha x & x \leq 0 \end{cases}$$

where $\alpha$ is a small positive constant like 0.01. its derivative is 1 for positive inputs and $\alpha$ for negative inputs, so the gradient never becomes zero completely. this fixes the dying relu issue.

another important activation is softmax, which is used for multi class classification. for a vector of logits $$z = (z_1, z_2, \dots, z_K)$$the softmax is

$$\text{Softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^K e^{z_j}}$$

this converts logits into a probability distribution over classes. the derivative is more complex since it’s a vector function, but the key thing is that it allows us to compute cross entropy loss easily for classification.
