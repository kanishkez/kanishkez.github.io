Normalization

Normalization is a way to stabilize activations/outputs in a neural network. Neural network can train poorly because:

  1. Activations can explode or vanish. This happens because during backpropagation, gradients are multiplied repeatedly by weights and derivatives of activation functions. If the derivative $f(z)$ is small (<1) repeated multiplication can make gradients shrink. Simiarly, if the gradients are big, repeated multiplication can make them huge causing them to explode.
  2. Internal Covariate Shift. It is a phenomenon where the distribution of inputs to the intermediate layers of a neural network changes during training because of the earlier layers' weights. At the start of the training, layer 5 may receive inputs with mean=0 and variance=1. After some weight updates, layer 5 may now receive inputs with mean=20 and huge variance. So, this forces deeper layers to continously readjust, slowing convergence and slowing down training. So, Normalization is used.

The core idea is that it stabilizes activations. $$\text{Normalized} = \frac{x - \mu}{\sigma}$$ It rescales values to a stable range, making optimization more predictable.

Modern networks also apply learnable scale and shift parameters, $\gamma$ and $\beta$:

$$\hat{x} = \gamma \cdot \frac{x-\mu}{\sigma} + \beta$$

Batch Normalization

It normalizes across the batch dimension/

For a minibatch $B = \{x_1, x_2, ..., x_m\}:$

We calculate the mean and standard deviation: $$\mu_B = \frac{1}{m} \sum_{i=1} x_i,\quad \sigma_B^2 = \frac{1}{m}\sum_{i=1}(x_i - \mu_B)^2$$ Then we normalize using that: $$\hat{x}_i = \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}}$$ FInally apply affine transform: $$y_i = \gamma \hat{x}_i + \beta$$

Normalization forces the data into a fixed distribution (mean=0,variance=1) That makes training stable but also restrict how much the network can learn.

Without affine parameters, every layer would only see standardized values limiting how much the network can learn. So affine parameters preserve model capacity.

$\beta$ controls the learned bias $\gamma$ controls learned feature scaling, how much the feature can increase or decrease

These are learnable parameters, that can decide how much the bias and scaling should be for a batch of features.

Used in CNNs.

Layer Normalization

It normalizes within a single sample, across a single layer.

Given a vector of activations for one token:

$$x = [x_1, x_2, ..., x_d]$$ We calculate mean and deviation: $$\mu = \frac{1}{d}\sum_{j=1} x_j,\quad \sigma^2 = \frac{1}{d}\sum_{j=1}(x_j - \mu)^2$$ Then we get the output as:

$$y_j = \gamma \frac{x_j - \mu}{\sqrt{\sigma^2 + \epsilon}} + \beta$$

It is independant of batch size.

It is used in Transformers, RNNs/LSTMs and models with varying sequence lengths.

RMSNorm

RMSNorm removes the mean from LayerNorm

Formula:

$$\text{RMS}(x) = \sqrt{\frac{1}{d} \sum_{j=1} x_j^2}$$ Output: $$y = \frac{x}{\text{RMS}(x)} \cdot \gamma$$ There is no subtracting mean or no $\beta$ shift

It is not centered around zero/mean.

It is simpler,lower compute and has better stability in large transformers

Used in LLaMA, Falcon, Mistral

Removing the subtracting from mean part can reduce computation, removes extra instability when mean fluctuates and in large networks like LLMs, centering around the mean isn't necessary.

So instead of forcing. zero mean, it just ensures that the mean or all inputs are stable.
