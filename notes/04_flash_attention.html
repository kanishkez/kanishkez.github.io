Flash Attention

Standard attention gets incredibly slow as sequence length grows, mostly because we keep shuffling huge matrices back and forth between GPU memory types. Flash Attention fixes this. It isn't a new attention architecture per se, but rather a way of utilizing GPU RAM and resources efficiently so as to make the attention process faster and more efficient.

In standard self attention: $$\text{Attention}(Q, K, V) = \text{softmax}\left( \frac{QK^T}{\sqrt{d}} \right) V$$ The intermediate matrix $QK^T$ has a shape of $N \times N$. For a sequence of let's say 4000 tokens, that is 16 million elements to compute. On modern GPUs, this is obviously very expensive.

Modern GPUs

The GPU memory hierarchy comprises memory of different sizes and speeds, with smaller memory being faster.

In an A100 GPU with 80 GB of memory, its on chip memory SRAM is only 192 KB per each of the 108 streaming multiprocessors. On chip memory SRAM is always faster than the off chip HBM memory but it is smaller in size. So what happens is that each kernel loads inputs from HBM to registers and SRAM, computes the operation (matrix multiplication in this case), and then stores it back on HBM.

This process of constantly moving data from HBM to SRAM is very inefficient. So, it is important to exploit SRAM more and not make the moving memory process a bottleneck.

How normal attention works:

  1. Loads Q and K from HBM to registers/SRAM.
  2. Computes $QK^T$ (using cuBLAS GEMM) and writes the entire matrix back to HBM.
  3. Loads that big matrix again to apply softmax.
  4. Writes the softmaxed attention weights back to HBM.
  5. Loads those weights again and multiplies by V.

This means that we need 3 memory round trips for one attention head.

Here is how Flash Attention changes this memory fetching logic and uses SRAM efficiently:

It breaks the key and value sequences into blocks called tiles (say 128 tokens at a time): $$K = [K_1, K_2, K_3, \dots], \quad V = [V_1, V_2, V_3, \dots]$$ Each of these tiles fits into SRAM.

So, for each query vector $q_i$, we process one K/V tile at a time:

We load tile $K_j, V_j$ from HBM to SRAM.

Compute scores: $$S_{ij} = q_i K_j^T$$ (stays in registers/SRAM)

Compute tile wise max and sum of exponentials: $$m_{ij} = \max(S_{ij}), \quad l_{ij} = \sum e^{S_{ij} - m_{ij}}$$

For the next tile, we update the running max and sum of exponentials: $$m_i^{\text{new}} = \max(m_i^{\text{old}}, m_{ij}), \quad l_i^{\text{new}} = e^{m_i^{\text{old}}-m_i^{\text{new}}} l_i^{\text{old}} + e^{m_{ij}-m_i^{\text{new}}} l_{ij}$$ where $l_i$ is the denominator in the softmax function.

Compute partial attention output for the tile: $$O_i^{(j)} = \sum e^{S_{ij} - m_{ij}} V_j$$

Accumulate this into the running output vector $O_i$: $$O_i^{\text{new}} = \frac{l_i^{\text{old}} e^{m_i^{\text{old}} - m_i^{\text{new}}} O_i^{\text{old}} + e^{m_{ij} - m_i^{\text{new}}} O_i^{(j)}}{l_i^{\text{new}}}$$

Once the tile finishes, the kernel loads the next tile from HBM into SRAM.

It keeps:

  • $q_i$
  • running $m_i, l_i$
  • running $O_i$ inside on chip memory.

So we only need to read/write the running output $O_i$ (which is $N \times d$) and load the current tile keys and values from HBM. We never write the massive $N \times N$ matrix to HBM.

We save time and memory by operating on shared memory and registers most of the time. Avoiding building $N \times N$ attention maps means there is way less time spent waiting for memory, so Flash Attention is way faster and more efficient.

Total traffic in naive attention is quadratic ($O(N^2)$) since it computes $QK^T$ and then writes it into HBM, then applies softmax, then multiplies it by V.

So total traffic kinda becomes $5 N^2$.

In Flash Attention:

We just load tiles of size say $B \times d$ and compute attention locally in SRAM.

We never write all of the attention matrix into HBM at once.

So total traffic becomes $O(N)$ (or more specifically $O(Nd + N^2 / M)$ where $M$ is the SRAM size, which is practically linear for typical sequence lengths).

So basically, making attention IO aware—which is the entire premise of Flash Attention—means we make the computation on GPUs efficient and fast by using the memory hierarchy correctly.

By keeping the most critical intermediate steps inside fast on chip SRAM and updating the softmax normalization factors on the fly, we bypass the huge memory bandwidth bottleneck. This IO aware design makes long context training and inference actually feasible instead of hitting a wall.
