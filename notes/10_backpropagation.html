Motivation

Backpropagation is the algorithm that enables efficient training of deep neural networks by computing gradients of the loss function with respect to all parameters. The motivation for backpropagation arises from the need to optimize complex, multi layered models with millions of parameters, where naive computation of gradients would be computationally prohibitive.

In ML, backpropagation is foundational, making it possible to train deep networks on large datasets. Its efficiency and scalability have driven the success of deep learning in a wide range of applications, from computer vision to natural language processing.

Chain Rule

The chain rule of calculus is the mathematical foundation of backpropagation. (See Calculus and Neural Networks chapters for more details)

Computational Graph

A computational graph is a directed acyclic graph that represents the sequence of operations in a neural network. Nodes correspond to operations or variables, and edges represent dependencies. The graph structure enables efficient forward and backward passes by organizing computations and dependencies.

In ML, computational graphs are used in automatic differentiation frameworks, enabling flexible model design and efficient gradient computation. Visualization of computational graphs aids in understanding model structure and debugging.

Forward Pass

The forward pass computes the output of the network by propagating the input through each layer. (See Neural Networks chapter for more details)

Backward Pass

The backward pass computes gradients of the loss function with respect to each parameter by propagating errors backward through the network. (See Neural Networks and Optimization chapters for more details)

Gradients

Gradients are vectors of partial derivatives that indicate the direction and rate of change of the loss function with respect to the parameters. (See Calculus and Optimization chapters for more details)

Parameter Update

Parameter updates are performed using the computed gradients, typically via gradient descent or its variants. (See Optimization chapter for more details)

Efficiency

Efficiency in backpropagation is achieved by reusing intermediate computations, exploiting the structure of the computational graph, and optimizing memory usage. Techniques such as dynamic computation graphs and operator fusion further improve efficiency.

In ML, efficient backpropagation enables the training of large, deep networks on modern hardware. Understanding and optimizing efficiency is essential for scaling models and reducing training time.

Vanishing/Exploding Gradients

Vanishing and exploding gradients are phenomena that occur when gradients become too small or too large during backpropagation, leading to slow or unstable training. These issues are common in deep or recurrent networks and can prevent effective learning.

In ML, techniques such as careful initialization, normalization, and specialized architectures (e.g., LSTM, ResNet) are used to mitigate vanishing and exploding gradients. Understanding these phenomena is essential for designing robust and scalable models.

Backpropagation: Complete Notes

Core Idea

Backpropagation computes gradients of the loss with respect to all parameters (weights, biases, filters) using the chain rule of calculus, allowing the model to learn via gradient descent.

Chain Rule Refresher with a Simple Example

Let:

  • $$z = f(x) = (3x + 2)^2$$

  • You want to compute $$\frac{dz}{dx}$$ Break it down:

  • Let $$u = 3x + 2$$

  • Then $$z = u^2$$ Apply the chain rule:

$$\frac{dz}{dx} = \frac{dz}{du} \cdot \frac{du}{dx} = 2u \cdot 3 = 2(3x+2) \cdot 3 = 6(3x+2)$$

This is the basis of how gradients are propagated layer by layer in neural networks.

Why We Need Gradient w.r.t. Input in Deep Neural Networks

In deep networks, each layer passes activations to the next. So during backprop, we need to:

  1. Compute the gradient of the loss with respect to the parameters (weights, biases) in each layer.

  2. To compute this, we need the gradient of the loss w.r.t. the inputs of that layer.

Key Rule:

To update parameters in a layer, say layer L, we need:

$$\frac{\partial \mathcal{L}}{\partial W^{(L)}} = \frac{\partial \mathcal{L}}{\partial a^{(L)}} \cdot \frac{\partial a^{(L)}}{\partial z^{(L)}} \cdot \frac{\partial z^{(L)}}{\partial W^{(L)}}$$

Where:

  • a^{(L)}: activation

  • z^{(L)}: input to activation function

  • W^{(L)}: weights

To compute $$\frac{\partial \mathcal{L}}{\partial a^{(L-1)}}$$which is needed to update earlier layers, we need:

$$\frac{\partial \mathcal{L}}{\partial a^{(L-1)}} = \frac{\partial \mathcal{L}}{\partial z^{(L)}} \cdot \frac{\partial z^{(L)}}{\partial a^{(L-1)}}$$

So gradients must be passed back through the inputs.

Example:

Take 2 layers in a Network

Consider the output as A

In Neuron 1 using weights W and input X we get: Y1=W1 * x + b1 This Y1 is then in turn used as an input in the next Neuron 2 So: Y2= W2 * y1 + b2

So in order to compute the gradient of the first Neuron 1 we need the input gradient of Y2 (which is y1) To update the weights of Neuron 1 (W_1), we need to compute:

$$\frac{\partial \mathcal{L}}{\partial W_1}$$ We apply the chain rule backwards:

$$\frac{\partial \mathcal{L}}{\partial W_1} = \frac{\partial \mathcal{L}}{\partial y_2} \cdot \frac{\partial y_2}{\partial y_1} \cdot \frac{\partial y_1}{\partial W_1}$$ So $$\frac{\partial \mathcal{L}}{\partial W_1} = \left( \frac{\partial \mathcal{L}}{\partial y_2} \cdot W_2 \right) \cdot x^\top$$

CNN Backpropagation Step by Step

Assume a 2D convolution with:

  • Input: $$X \in \mathbb{R}^{H \times W}$$

  • Filter: $$K \in \mathbb{R}^{k \times k}$$

  • Output: $$Y = X * K \in \mathbb{R}^{(H-k+1) \times (W-k+1)}$$

Here, * denotes cross correlation (not convolution).

Forward Pass (Cross Correlation)

$$Y[i, j] = \sum_{m=0}^{k-1} \sum_{n=0}^{k-1} X[i + m, j + n] \cdot K[m, n]$$

This slides the filter over the input, performing elementwise multiplication and summing the result.

Backpropagation in CNNs

Let’s compute:

  1. $$\frac{\partial \mathcal{L}}{\partial K}$$Gradient w.r.t. the filter
  2. $$\frac{\partial \mathcal{L}}{\partial X}$$ Gradient w.r.t. the input

Let $$\frac{\partial \mathcal{L}}{\partial Y}$$= $$\delta_Y$$(the upstream gradient).

1. Gradient w.r.t. Filter (Kernel)

We want:

$$\frac{\partial \mathcal{L}}{\partial K[m, n]} = \sum_{i,j} \frac{\partial \mathcal{L}}{\partial Y[i,j]} \cdot \frac{\partial Y[i,j]}{\partial K[m,n]}$$

From:

$$Z_{i,j} = \sum_{m’=0}^{f-1} \sum_{n’=0}^{f-1} X_{i+m’, j+n’} \cdot K_{m’,n’}$$

To get:

$$\frac{\partial Z_{i,j}}{\partial K_{m,n}} = X_{i+m, j+n}$$

because K_{m,n} appears only once in that sum (as long as m’,n’ are indexed uniquely). This is because we take gradients for each parameter seperately. The kernel $$K \in \mathbb{R}^{f \times f}$$is a matrix of learnable parameters.

When doing backprop, you compute:

$$\frac{\partial \mathcal{L}}{\partial K_{m,n}} \quad \text{for all } m,n$$

Each entry $$K_{m,n}$$is treated as its own variable.

So:

  • When we compute:
$$\frac{\partial \mathcal{L}}{\partial K_{0,1}} = \sum_{i,j} \delta_{i,j} \cdot X_{i+0,j+1}$$
  • That’s just one of many such computations — we’ll also compute:
$$\frac{\partial \mathcal{L}}{\partial K_{1,0}},\ \frac{\partial \mathcal{L}}{\partial K_{1,1}}, \text{ etc.}$$

This is why in the gradient expression:

$$\frac{\partial \mathcal{L}}{\partial K_{m,n}} = \sum_{i,j} \delta_{i,j} \cdot X_{i+m, j+n}$$ $$Y[p, q] = \sum_{m,n} K[m,n] \cdot X[p + m, q + n]$$

So:

$$\frac{\partial Y[i,j]}{\partial K[m,n]} = X[i + m, j + n]$$

Thus:

$$\frac{\partial \mathcal{L}}{\partial K[m,n]} = \sum_{i,j} \delta_Y[i,j] \cdot X[i + m, j + n]$$

This is equivalent to cross correlating the input with the gradient.

2. Gradient w.r.t. Input

We want:

$$\frac{\partial \mathcal{L}}{\partial X[i,j]} = \sum_{p,q} \frac{\partial \mathcal{L}}{\partial Y[p,q]} \cdot \frac{\partial Y[p,q]}{\partial X[i,j]}$$

From forward pass:

$$Y[p,q] = \sum_{m,n} X[p + m, q + n] \cdot K[m,n]$$

So:

$$\frac{\partial Y[p,q]}{\partial X[i,j]} = \begin{cases} K[i - p, j - q] & \text{if } (i - p, j - q) \in [0,k-1] \ 0 & \text{otherwise} \end{cases}$$

So:

$$\frac{\partial \mathcal{L}}{\partial X[i,j]} = \sum_{m,n} \delta_Y[i - m, j - n] \cdot K[m,n]$$

This is equivalent to a full convolution of \delta_Y with the rotated kernel.

\frac{\partial \mathcal{L}}{\partial K}\delta_Y * XCross correlation of input with upstream grad
\frac{\partial \mathcal{L}}{\partial X}\delta_Y * K^{\text{rot}}Convolution with rotated kernel
\frac{\partial \mathcal{L}}{\partial Y}Comes from next layerUsed to compute both above

Final Takeaways

  • Backprop relies on chain rule, computing gradients layer by layer.

  • To update weights in a layer, we need loss gradient w.r.t. that layer’s input.

  • In CNNs:

  • Gradients wrt filter = cross correlate input with upstream gradient

  • Gradients wrt input = convolve upstream gradient with rotated kernel

  • Only one term survives in convolution derivatives due to alignment — crucial for efficiency.
