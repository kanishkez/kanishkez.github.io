Transformers

Honestly, transformers are the backbone of modern deep learning, completely replacing recurrent networks by processing everything in parallel. Instead of going step by step, they rely on self attention to figure out which tokens actually matter to each other. Here is my breakdown of how the whole encoder decoder setup actually works under the hood.

The main concept of transformers is self attention which chooses relevant important tokens when inputting a token.

Stages:

Encoder:

  1. Conversion of input to embedding and positional encoding
  2. Multi head attention while inputting tokens
  3. Add & Norm
  4. Fully connected layers
  5. Add & Norm Decoder:
  6. Self Masked Multi Head attention
  7. Add & Norm
  8. Multi head attention with encoder output
  9. Add & Norm
  10. Feed forward layer
  11. Linear layer
  12. Softmax probabilities

Encoder

Embeddings

An embedding is a learned numerical representation of a word or a token. Instead of just writing 'how' as 101, we write it as $[0.25, -0.19, 0.56, 0.04, \dots]$ so that similar words have similar values.

Positional Encoding

Transformers have no inherent notion of sequence order. To provide this information, positional encoding is introduced.

Transformers process input tokens in parallel, unlike RNNs or CNNs. Hence, they have no built in idea of token order. So positional encodings are used for the model to understand how certain words come after other words and to form actual sentences in which the positioning of words matters.

Why not use scalars for each position?

Although in the sentence 'I love machines', 'I' comes first and so on, in another sentence 'I' may actually be used somewhere else. Scalars do not do justice when the model has to understand the positioning of tokens in high dimensions. Scalars would only lead to a linear understanding of positioning.

Why Sinusoidal Functions?

Sinusoidal functions are continuous, periodic, and allow for unique vector representations. They encode both absolute and relative position information. Using multiple frequencies allows different dimensions to capture different positional granularities.

We define:

  • Wavelength ($\lambda$): Distance between two peaks. Frequency $f = \frac{1}{\lambda}$.
  • Frequency: How many oscillations a function completes in unit time.
  • Even though time is not explicit here, the sequence index acts as a surrogate for it.

Positional Encoding Formula

Let $d_{\text{model}}$ be the model's embedding size. For a given position $pos$ and dimension $i$:

$$PE_{(pos, \, 2i)} = \sin\left(\frac{pos}{10000^{\frac{2i}{d_{\text{model}}}}}\right)$$ $$PE_{(pos, \, 2i+1)} = \cos\left(\frac{pos}{10000^{\frac{2i}{d_{\text{model}}}}}\right)$$ where $$w_{i} = \frac{1}{10000^{\frac{2i}{d_{\text{model}}}}}$$ where $d_{\text{model}}$ is the embedding dimension (how many numerical representations/numbers used for 1 token, usually 512) and $i$ represents the dimension index within each token's embedding.

Why sine and cosine functions to represent positions?

They keep the values between 0 and 1. Most importantly, sine and cosine capture relative positioning. If you know the position of a token, then the token that is $k$ steps away from that token can also be found using the formulas:

$$\sin(x + \delta) = \sin(x)\cos(\delta) + \cos(x)\sin(\delta)$$ $$\cos(x + \delta) = \cos(x)\cos(\delta) - \sin(x)\sin(\delta)$$ where $\delta$ is how far the token is from the token $x$ which we already have.

Suppose we have an encoding at position $pos$: $$v_{\text{pos}} = \begin{bmatrix} \sin(\alpha) \\ \cos(\alpha) \end{bmatrix}$$ then the encoding at $pos+k$ is: $$v_{\text{pos}+k} = \begin{bmatrix} \sin(\alpha + \beta) \\ \cos(\alpha + \beta) \end{bmatrix} = \underbrace{\begin{bmatrix} \cos(\beta) & \sin(\beta) \\ -\sin(\beta) & \cos(\beta) \end{bmatrix}}_{\text{Rotation Matrix}} \cdot \begin{bmatrix} \sin(\alpha) \\ \cos(\alpha) \end{bmatrix}$$ where $\beta = k \cdot w_{i}$.

The positional encoding at position $pos+k$ is just a rotation of the encoding at $pos$ by an angle $\beta = k \cdot w_{i}$. This means that the model can learn a consistent notion of relative position without any parameters.

Integration into Transformer Attention

Let the token sequences be: $$x = [x_1, x_2, \dots, x_n]$$ Each token is embedded: $$\text{Embed}(x_i) = E[x_i]$$ Now to this embedding we add $x_i$'s positional encoding, so the modified $x_i$ becomes: $$\tilde{x}_i = \text{Embed}(x_i) + \text{PE}(i)$$ The positional encodings allow $Q$ and $K$ to now contain both semantic and positional information, allowing the attention scores to factor in sequence order.

Self Attention

Input vector is turned into 3 vectors which provide information/context about the input and what other vectors are important/relevant to this specific input:

  1. Query: This vector collects information from what is needed for the input vector. It is a question asking what information is relevant to the current input vector.
  2. Key: All vectors have a key vector which is used to match the input's query vector with. So for a given query vector from an input, we look for the answer (the matching information) in every other vector's key vector. The key vector helps locate relevant positions. Value is the one that contains the meaning of those vectors. The key vector does not carry the position, it just carries a compressed version of the token's semantic role in the attention mechanism. The key represents what information is held by the token and not the information itself. Note: Even the input vector has a key vector that is trained during training. This is how all other vectors build up their key vector.
  3. Value: After we find the necessary answer to the query, this vector actually carries the value (the meaning of every word/token) in it. It actually holds the information of tokens that means something. It has the same source as the Key vector.

These 3 vectors are trained through weights and are linear projections of the input vector that is fed.

For each input token $x_{i}$: $$q_i = W^Q x_i,\quad k_i = W^K x_i,\quad v_i = W^V x_i$$ Where the dimensions are: $$W^Q, W^K, W^V \in \mathbb{R}^{d_k \times d_{\text{model}}}$$

Attention

$$\text{Scores} = QK^T$$ This dot product tells us the relationship between query $Q$ and key $K$ and how they are related to each other. This tells us how important $Q$ is to $K$.

This gives a sequence length $\times$ sequence length matrix which tells us for the given query what other keys mean to it. The dot product of the query vector and other key vectors helps us understand what tokens are important and relevant to our input vector.

$$\text{Scaled Scores} = \frac{QK^T}{\sqrt{d_k}}$$

How the matrix looks: Each query is represented by a row. Each key is represented by a column. So the queries in all rows 1 to $n$ are being compared to keys in all columns 1 to $n$.

Say, at position 2,3 we are comparing how the second query is affected by the third key. We divide the scores by the square root of the dimensions to avoid large values.

$$\text{Attention Weights} = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)$$

Then multiply Values to the aggregate: $$\text{Output} = \text{Attention Weights} \cdot V$$ Each output has a weighted sum of value vectors of all tokens (their meaning) times the amount of attention paid to each of them.

So what is actually happening here is for all the Values of every token (what every token means) we are actually only taking the relevant and important information to our specific input by multiplying all the values with the attention weights which scale the value vectors to our need to suit the input. (that's a big ass line btw)

What is happening in Self Attention: The input query vector wants to find tokens that are important/relevant to itself. So when the Key and Query dot product is trained and the output is high, it means that for that specific query the key vector is really important. So we need to fetch this key vector's value (meaning) because it is important for the context.

Example: He went to the bank to withdraw money. When attending to the word “withdraw”, the model might:

  • Assign a high attention weight to “bank”,
  • Because the Query(“withdraw”) ⋅ Key(“bank”) is high,
  • And then it pulls in Value(“bank”) — i.e., semantic information about financial institutions.

The model learns in training to adjust its matrices in such a way that it gives high values to key query dot products that are relevant. When it comes to the value vector, in training it is adjusted such that the key which has a higher importance, its value also has a heavier weight which makes sure that this value comes out on top despite the values of all tokens being used. So in this case, the value of 'bank' will come out on top because it will have a higher numerical value. The model trains itself that way.

Multi Head Attention

The above process is for one head as in one iteration. We use multiple such heads (kind of like using many CNNs so that each CNN layer learns multiple features) so that different heads can focus on different aspects of the input.

We compute $h$ heads in parallel:

  1. Calculate Query, Key, Value
  2. Calculate attention weights and scale values by attention weights
  3. Get the output

For $h$ attention heads we compute: $$\text{head}_i = \text{Attention}(Q_i, K_i, V_i) = \text{softmax}\left( \frac{Q_i K_i^\top}{\sqrt{d_k}} \right) V_i$$

Combining the heads

After all the heads are computed, we concatenate them so that all the relevant features are combined. $$\text{Concat}(\text{head}_1, \text{head}_2, \dots, \text{head}_h)$$

All of these are then passed through a final linear layer: $$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h)W^O$$ where $W^O \in \mathbb{R}^{h d_v \times d_{\text{model}}}$ is a learnable projection matrix.

After Multi Head Attention

We add a Residual Connection: $$\text{out}_1 = \text{LayerNorm}(\text{input} + \text{MultiHead}(Q, K, V))$$ This ensures gradient flow and helps preserve original information as well as stabilizes training.

Feed Forward Networks

This is a dense layer applied to each token: $$\text{FFN}(x) = \text{ReLU}(xW_1 + b_1)W_2 + b_2$$ This is done to introduce non linearity to the model so that it learns more complex patterns and extracts more features from data.

After this we add another Residual Connection and LayerNorm: $$\text{out}_2 = \text{LayerNorm}(\text{out}_1 + \text{FFN}(\text{out}_1))$$ That is the Encoder block complete.

Decoder

Decoder block is where the output part generation happens. Whether it is text prediction or text translation or whatever, this is where that part happens.

Masked Multi Head Attention

The decoder starts with using Multi Head attention to generate outputs given inputs. In Masked Multi Head attention we do something called shift right where in training we push the output prediction tokens one token to the right so that the model learns instead of seeing all the correct predictions.

Example:

Decoder InputDecoder Output
[ start token ]I
Ilove
lovecats
cats[end token]

This is done so that during training we create perfect pairs. For example, if we give the start with the input 'I' itself without giving a start token, then in the first token generation it will have to do is 'love' and it will entirely miss the 'I' token. So we shift the target tokens right so that teacher forcing is feasible. So when we start with the [start token] we can teacher force it starting with 'I'.

Masked Attention

While training the attention in the decoder, we hide the future outputs from the current input state since it is going to predict the future outputs.

For example: For the sentence 'I am a swimmer' if we are at the word 'a', we hide the key/value vector of 'swimmer' from 'a' because 'swimmer' has not been generated yet and to show the word 'swimmer' would affect the model learning and could be considered cheating.

Each token at $t$ can only see tokens before it till $t-1$ and not the tokens ahead of it.

For a decoder input sequence: $$Y = [y_1, y_2, \dots, y_T]$$ We compute: $$Q = Y W^Q$$ $$K = Y W^K$$ $$V = Y W^V$$ In calculating the attention however: $$\text{Attention}(Q, K, V) = \text{softmax} \left( \frac{QK^\top}{\sqrt{d_k}} + M \right) V$$ Mask $M$ is used.

$M$ is a matrix of shape $T \times T$ where entries above the diagonal are set to $-\infty$ and entries on or below the diagonal are 0.

Suppose we take $T=4$: $$\text{Mask} = \begin{bmatrix} 0 & -\infty & -\infty & -\infty \\ 0 & 0 & -\infty & -\infty \\ 0 & 0 & 0 & -\infty \\ 0 & 0 & 0 & 0 \end{bmatrix}$$ Here at row 0 it can only attend to column 0 and not columns ahead of it. At row 1 it can attend to 0 and 1 only and not columns ahead of it, and so on. So adding this matrix to the soft score means that the $-\infty$ is added to the values greater than the current sequence's values, nullifying their existence.

What's happening at this stage?

Input Queries taken are the previously generated outputs by the decoder. All tokens generated so far are taken as queries. Keys used are the previously generated output tokens.

At each position $t$, we compute how much attention it should pay to each earlier token $< t$, in order to build a contextualized understanding of what has been generated so far.

Masked self attention in the decoder is the mechanism that enables the model to “think” about the past before generating the next token — while strictly preventing cheating by looking ahead.

The Query at position $t$ asks: “What do I need to know from the past to decide what to generate next?” The Keys and Values contain the representations of all previous tokens.

The model is basically building up context from what has been generated so far and assigning weights to every token with respect to every other token (that has been generated so far ONLY past tokens not future tokens) to see what needs to be generated in the future.

A good intuition is that when you are speaking and framing a sentence 'I am Kanishk and I am doing-' here the word 'I' refers to Kanishk and the second 'I' also refers to him and what he is doing. This information is inferred through previously spoken words. What you say next depends on what you have said before. That is what is happening in this layer.

After this we add residual connections and layer norm.

Encoder Decoder Multi Head Attention

This attention module lets the decoder view the encoder's outputs and acts upon them. The first attention block focuses on what the decoder has generated already, while this one focuses on the encoder's outputs and how they can be used to generate the next token.

Input

  • Queries: Decoder hidden states from the previous layer
  • Keys and Values: Encoder hidden states (final outputs of the encoder)

Output

Context vectors that explain which encoder vectors are important to which decoder vectors. What information to infer from the encoder to produce the next token.

This step is called cross attention where we connect the source sentence with the target.

Same as any attention module:

  1. We create linear projections that serve as queries, keys and values.
  2. Assign attention scores using the formula: $$\text{Attention}(Q, K, V) = \text{softmax}\left( \frac{QK^T}{\sqrt{d_k}} \right) V$$ where encoder outputs serve as keys and values, and decoder output from the previous attention module serves as queries.
  3. So once the attention module finds keys that are relevant, their values are used to calculate attention.

Obviously since it is a multi head attention module, this process will happen for several heads so that all of them can capture different features and different information.

Everything about this block is the same as any multi head attention so there is not much to talk about here. What is important is why this block exists and what it does.

What's actually happening?

  1. This attention module ultimately puts together vectors from the encoder outputs that are important for the next token generation.
  2. It does so by finding out which tokens are important to the input decoder tokens and assigning heavy weights to their values.

So for example:

  • Encoder Output: Kanishk is going to cycle.
  • Decoder Output: He is 'cycling'.

If the model were to complete the sentence from the word 'is', it would look at the encoder output token 'cycle' and then give importance to it so that it can decide the next token 'cycling'.

Feed Forward Network

Each token goes through a two layer feed forward network. It has a linear layer, ReLU activation, followed by another linear layer:

  1. Linear layer: Increase dimensions from 512 to 2048
  2. ReLU activation (to add non linearity to the model)
  3. Linear layer: Project back to 512

This is applied separately at each timestep so each token position is processed independently.

Residual Connection and LayerNorm

Each sub layer (including the feed forward network and the encoder decoder attention module plus the self attention module) is wrapped in $\text{LayerNorm}(x + \text{SubLayer}(x))$. Basically, LayerNorm is applied after a residual connection is applied to the sub layers. This is to stabilize training and help the gradients flow better.

Context Vector

After all of the attention modules plus residual connections and layer norms plus linear layers and activation functions, the final context vector that we have is the representation of what the next token in the sequence is likely to be.

Everything we have done so far produces a context vector that shows us what the next token in the sequence could be. We are going to now use this context vector to find the next token.

Prediction of Tokens

The actual prediction of tokens happens in the linear layer + softmax module.

The decoder's output is a sequence of contextual vectors: $$\text{Decoder Output: } h \in \mathbb{R}^{T \times d_{\text{model}}}$$

We need to convert this into logits for every word in the vocabulary (predictions for every word in the vocabulary being the next token), so we apply a linear layer: $$\text{Logits} = h W^T + b$$ where $W \in \mathbb{R}^{V \times d_{\text{model}}}$ consists of learnable weights, and $b \in \mathbb{R}^V$ is the bias vector.

For every token $t$, you get a vector representing scores for every token in the vocabulary.

How prediction happens:

The weight matrix here consists of the entire vocabulary of tokens. Each row represents a token.

  • $W \in \mathbb{R}^{V \times d_{\text{model}}}$ — the weight matrix in the final linear layer.
  • $\mathbf{h}_t \in \mathbb{R}^{d_{\text{model}}}$ — the context vector from the decoder.
  • Hence the dimensions match for every row in the weight vector to be compared to the entire context vector.
  • Note: In many models this output weight matrix is actually tied to the input embedding matrix so that the model has one shared representation space for both understanding words and generating them.

If the context vector matches a row (through the dot product) then its logit score would be high. So we look for how similar the context vector and each token row in the weight matrix is and assign logits according to that.

Each dot product tells us how aligned the context is with each word's representation. A high dot product means that the current context vector is very similar to that word vector. This linear layer essentially compares the context with every possible word in the vocabulary.

Then we use softmax to get probabilities: $$P(y_t = w_i \mid \text{context}) = \frac{\exp(\text{logit}_i)}{\sum_{j=1}^{V} \exp(\text{logit}_j)}$$

The most probable token is the one whose embedding is most aligned (has the highest dot product) with the current context.

So that is the whole pipeline from raw input embeddings to the final probability distribution over the vocabulary. Once the model picks the best token, it feeds it back into the decoder to generate the next one, repeating the cycle. It is simple math when you break it down, but the parallelization makes it incredibly powerful.
