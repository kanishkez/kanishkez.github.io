LSTM and GRU

Table of Contents

RNN has a bunch of drawbacks that include:

  1. Exploding/Vanishing gradients
  2. No memory control- This means that the most recent memory is the one that is typically used the most to update the current one. For example, if I need to borrow memory from a whole other instance, I won't be able to do that. 'kanishk is french, he speaks french' Here the model can predict the word 'french' only if it looks at the previous memory where it is mentioned that the person is french So we need a way of controlling how much memory and which memory is used in the current instance. Here is where LSTM comes in, it stands for Long short term memory.

LSTM introduces a memory cell which has a bunch of gates that do the following funcitonalities:

Cell State

Cell states in lstm is like a passage running through the entire gates that exist, any information that passes through the network passes through the cell state The cell state consists of:

Forget Gate

It learns to forget the useless past memory from the past using the previous hidden and the current input So it decides what information to keep and what to forget based on the previous cell state and the new input information

$$f^{(t)} = \sigma(W_f x^{(t)} + U_f h^{(t-1)} + b_f)$$ where we use the sigmoid function to control the amount of information that needs to be added sigmoid function is used in gates to control the amount of information that flows through in lstms

The forget gate coupled with the previous cell state is added to the new input gate's dot product with the new candidate memory to update the entire memory

Input Gate

Input gate again decides how much of the new candidate memory content to be added

In a way input gate and forget gate both decide how much to add and forget to the memory respectively The same way that the forget state decides how much memory to be removed from the previous cell state, input gate decides how much of the new input should be added to the new current state $$i^{(t)} = \sigma(W_i x^{(t)} + U_i h^{(t-1)} + b_i)$$

Candidate Memory

This contains the new memory content, it contains the actual new information that might be stored in the memory

$$\tilde{c}^{(t)} = \tanh(W_c x^{(t)} + U_c h^{(t-1)} + b_c)$$ even though input gate and candidate memory have the same equation, the activation function plays a vital role in determining what their individual role is

Input gate decides how much of the information from candidate memory to let through

Update Memory(cell state)

$$c^{(t)} = f^{(t)} \circ c^{(t-1)} + i^{(t)} \circ \tilde{c}^{(t)}$$ here is where the memory update process happens the forget state is multiplied with the old state so that the memory is forgotten and that memory is added with the new memory that we're bringing in through the candidate memory and adjusting the amount with the help of input gate

Output gate

$$o^{(t)} = \sigma(W_o x^{(t)} + U_o h^{(t-1)} + b_o)$$ output gate again chooses how much of the cell state is to be used and sent to the hidden state

Hidden state

$$h^{(t)} = o^{(t)} \circ \tanh(c^{(t)})$$ Hidden state is the final output we pass on to the next layer, output gate decides how much of cell state is to be considered while computing the hidden state

GRU

GRU tries to achieve the same thing LSTU does but using less gates and less computational power

It only uses two gates in all:

Update Gate

$$z^{(t)} = \sigma(W_z x^{(t)} + U_z h^{(t-1)} + b_z)$$ Decides what percentage of past memory to keep vs how much to embrace the new info (in the final state equation) if z=0 then only past memory is kept, if z=1 then only new memory is kept

Reset Gate

$$r^{(t)} = \sigma(W_r x^{(t)} + U_r h^{(t-1)} + b_r)$$ controls how much of the paste state to use when learning new info low r, suppress parts of the old memory while learning input high r, consider full memory to keep while learning new input

Note: Even though update and reset gate use the same equations their purpose is completely different because of how their weights get trained in training.

Candidate Hidden state

$$\tilde{h}^{(t)} = \tanh(W_h x^{(t)} + U_h (r^{(t)} \circ h^{(t-1)}) + b_h)$$
  • This is the new state computed from current input
  • Notice: it uses the reset gate to selectively erase past info before mixing it into the candidate
  • This gate uses reset gate to erase the past while simultaneously taking new info inside
  • this new state is the entire info that 'could' be accepted into the network (it is like a draft of what memory could be given)
  • the next state, i.e the final hidden state, actually decides how much of this new info to keep and how much of the old info to keep

Final Hidden State

$$h^{(t)} = (1 - z^{(t)}) \circ h^{(t-1)} + z^{(t)} \circ \tilde{h}^{(t)}$$

this state actually decides how much mixture of the new info + old info to keep z decides the % of both the info that is why we use 1 - z and z so that both of them add upto 1 so that both of their percentages add upto 100% this is the actual memory update

note about activation functions used:

sigmoid:

sigmoid is used to control how much information or how much of a memory to accept since it lies in the range of (0,1) 0 would mean nothing to keep and 1 would mean to preserve the memory It is used in gates for the same reason, since gates decide how much of a memory they want to actually keep

tanh cant be used here since its range lies between (-1,1) which means it could invert or modify the data but thats not what gates are for, theyre for controlling how much data is allowed through

tanh:

it is used to process the information,content so we use both negative and positive values to show negative influence and positive influence for example, a word may have negative or positive influence on another word which tanh allows to show
