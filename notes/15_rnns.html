Recurrent Neural Networks

Recurrent Neural Networks are a class of neural networks designed to model sequential data. Unlike feedforward networks which assume inputs are independent, RNNs introduce temporal memory. They retain information about previous inputs in the sequence.

Sequential Data

Data in which order matters. For example:

  • Words in a sentence, text, video, and audio data.

Sequential Processing

Instead of taking all inputs at once, RNNs process one element of the sequence at a time, maintaining a hidden state that acts as memory.

Core Equation

At each time step $t$, the RNN updates a hidden state $h_t$: $$h_t = \tanh(W_{hx} x_t + W_{hh} h_{t-1} + b_h)$$

Then it outputs: $$y_t = W_{hy} h_t + b_y$$

Where:

  • $x_t$: input at time $t$
  • $h_{t-1}$: hidden state from the previous time step
  • $W_{hx}$: input to hidden weight matrix
  • $W_{hh}$: hidden to hidden weight matrix
  • $W_{hy}$: hidden to output weight matrix
  • $\tanh$: activation function nonlinearity
  • $y_t$: output from a layer or neuron

Usage of Tanh as an Activation Function

$$\tanh(x) \in (-1, 1)$$
  1. It outputs between $-1$ and $1$.
  2. It allows the model to understand positive and negative influence in data.
  3. Its outputs are centered around zero which leads to stable training and gradients.
  4. Tanh helps mitigate the vanishing gradient problem to an extent because the derivative of tanh is higher than sigmoid, and it solves exploding gradients to an extent because unlike ReLU it does not let everything in.
  5. We need to preserve both positive and negative values.

Although it can still suffer with vanishing gradients problem.

Retaining Memory

Even though in standard neural networks we take the output from one layer and send it to the other layer, they do not retain any memory of what was once in them or what has passed through them. They only take input and give output. In RNNs, layers borrow memory or hidden states from themselves during the previous point in time.

In RNNs, we deal with two axes:

  1. Timestamps: Data from each word (weight multiplied by input plus bias) is stored in a neuron for future use. Words are processed one by one in order. Every word has a time step which consists of all the layers at once. The word is sent to all the layers and that is considered one time step.
  2. Layers: The depth of the model with many layers, like in any neural network. Layers are stacked vertically, and output from one layer goes to input as the next layer in the same time step. Each layer has its own hidden state.

Example: Let us say you have a sentence: "I love pizza" which is represented as $[x_1, x_2, x_3]$. We break it into three words:

Time StepWord
$t=1$I
$t=2$love
$t=3$pizza

Consider that we are using two layers in the model.

At Timestep One

  1. Layer 1: It receives the word 'I', computes a hidden state, and using that hidden state computes an output to pass on to the next layer.
  2. Layer 2: It receives the output from Layer 1, builds a hidden state, stores the data, and then uses that to get an output.

At Timestep Two

  1. Layer 1: It receives the word 'love' and remembers the hidden state from time step one about the word 'I'. It then computes the hidden state using the previous hidden state. This hidden state is then used to get an output and send to the next layer.
  2. Layer 2: It receives the input from Layer 1, uses its previous hidden state from time step one, and gets an output using it.

So in this way, time steps are used to remember the previous word data so that we can find meaning between all the words and relate the words. Not using a timestamp would mean that we process all the words separately without any relationship or meaning between them.

The neuron uses its previous hidden state data in a forward pass.

Sharing Weights and Parameters

In RNNs, a neuron or layer shares the same weights across the time steps. We do not want the model to learn different parameters for each time step so that the model learns the meaning of the entire sentence using all the words.

Backpropagation

1. Forward Pass in RNNs

%TOKEN_LINE_115%Given a sequence input $x^{(1)}, x^{(2)}, \dots, x^{(T)}$, the RNN computes: $$\begin{align*} h^{(t)} &= \tanh(W_{hh} h^{(t-1)} + W_{xh} x^{(t)} + b_h) \\ y^{(t)} &= W_{hy} h^{(t)} + b_y \end{align*}$$

Where:

  • $W_{xh} \in \mathbb{R}^{n \times d}$ is the input to hidden weight matrix.
  • $W_{hh} \in \mathbb{R}^{n \times n}$ is the hidden to hidden recurrent weight matrix.
  • $W_{hy} \in \mathbb{R}^{m \times n}$ is the hidden to output weight matrix.
  • $h^{(0)} = 0$ is the initial hidden state, which can also be a learned parameter.

Loss is computed at each time step: $$\mathcal{L} = \sum_{t=1}^T \mathcal{L}^{(t)}$$

Each $\mathcal{L}^{(t)}$ is typically cross entropy or mean squared error comparing $y^{(t)}$ with target $\hat{y}^{(t)}$.

2. Why We Need Gradient w.r.t Input

Each hidden state $h^{(t)}$ depends on the previous hidden state $h^{(t-1)}$. So when you compute the gradient of the loss with respect to the weights $W_{hh}$, you need: $$\frac{\partial \mathcal{L}}{\partial W_{hh}} = \sum_{t=1}^{T} \frac{\partial \mathcal{L}}{\partial h^{(t)}} \cdot \frac{\partial h^{(t)}}{\partial z^{(t)}} \cdot \frac{\partial z^{(t)}}{\partial W_{hh}}$$

where $z^{(t)} = W_{hh} h^{(t-1)} + W_{xh} x^{(t)} + b_h$. Clearly here $z^{(t)}$ derived with respect to $W_{hh}$ will give $h^{(t-1)}$ because the previous hidden state is used to compute $z$, and $h^{(t)} = \tanh(z^{(t)})$ which is the final output.

$$\frac{\partial z^{(t)}}{\partial W_{hh}} = h^{(t-1)}$$

So we need the gradient of the loss w.r.t. each hidden input $h^{(t)}$. This forms the basis of Backpropagation Through Time (BPTT).

3. The Core Recurrence in Backpropagation

The total gradient of the loss w.r.t. $h^{(t)}$ is: $$\frac{\partial \mathcal{L}}{\partial h^{(t)}} = \frac{\partial \mathcal{L}^{(t)}}{\partial h^{(t)}} + \frac{\partial \mathcal{L}}{\partial h^{(t+1)}} \cdot \frac{\partial h^{(t+1)}}{\partial h^{(t)}}$$

Why We Add and Not Multiply $h^{(t)}$

$h^{(t)}$ affects both its own time step loss $\mathcal{L}^{(t)}$ and all future losses through $h^{(t+1)}$. Each path contributes a term to the gradient, so we add them. This is the chain rule over time, not a product of losses.

4. Unrolling the Recurrence

Each $h^{(t+1)}$ is a function of $h^{(t)}$: $$\begin{align*} z^{(t+1)} &= W_{hh} h^{(t)} + W_{xh} x^{(t+1)} + b \\ h^{(t+1)} &= \tanh(z^{(t+1)}) \end{align*}$$

So: $$\frac{\partial h^{(t+1)}}{\partial h^{(t)}} = \text{diag}(1 - (h^{(t+1)})^2) \cdot W_{hh}$$

5. Final Expression

$$z^{(t)} = W_{hh} h^{(t-1)} + W_{xh} x^{(t)} + b_h$$ $$h^{(t)} = \tanh(z^{(t)})$$

So: $$\frac{\partial h^{(t)}}{\partial z^{(t)}} = 1 - (h^{(t)})^2 \quad \text{(element wise)}$$

Then to compute: $$\frac{\partial \mathcal{L}}{\partial W_{hh}} = \sum_{t=1}^{T} \frac{\partial \mathcal{L}}{\partial h^{(t)}} \cdot \frac{\partial h^{(t)}}{\partial z^{(t)}} \cdot \frac{\partial z^{(t)}}{\partial W_{hh}}$$

where: $$\frac{\partial z^{(t)}}{\partial W_{hh}} = h^{(t-1)}$$

For each time step: $$\frac{\partial \mathcal{L}}{\partial W_{hh}} += \delta^{(t)} \cdot (h^{(t-1)})^T$$

where: $$\delta^{(t)} = \frac{\partial \mathcal{L}}{\partial h^{(t)}} \circ (1 - (h^{(t)})^2)$$

To compute gradient at time $t$: $$\frac{\partial \mathcal{L}}{\partial h^{(t)}} = \frac{\partial \mathcal{L}^{(t)}}{\partial h^{(t)}} + (W_{hh}^T \cdot \delta^{(t+1)})$$

Compute: $$\delta^{(t)} = \left( \frac{\partial \mathcal{L}^{(t)}}{\partial h^{(t)}} + W_{hh}^T \delta^{(t+1)} \right) \circ (1 - (h^{(t)})^2)$$

6. Why Losses are Added

We add losses: $$\mathcal{L} = \sum_{t=1}^{T} \mathcal{L}^{(t)}$$

Even though the model might be predicting sequence likelihood as a product: $$P(y^{(1)}, \dots, y^{(T)}) = \prod_{t=1}^T P(y^{(t)})$$

When we apply negative log, the total loss becomes: $$\mathcal{L} = -\sum_{t=1}^{T} \log P(y^{(t)})$$

So even when the model multiplies probabilities, the loss adds due to log.

7. Numerical Example: Three Timestep RNN

Let us say:

  • Input dimension: 2
  • Hidden dimension: 2
  • Sequence length: 3

Initial weights: $$W_{xh} = \begin{bmatrix} 0.1 & 0.3 \\ 0.2 & 0.4 \end{bmatrix}, \quad W_{hh} = \begin{bmatrix} 0.5 & -0.1 \\ 0.6 & 0.2 \end{bmatrix}, \quad W_{hy} = \begin{bmatrix} 1.0 & -1.0 \end{bmatrix}$$

Inputs: $$x^{(1)} = \begin{bmatrix} 1 \\ 0 \end{bmatrix}, \quad x^{(2)} = \begin{bmatrix} 0 \\ 1 \end{bmatrix}, \quad x^{(3)} = \begin{bmatrix} 1 \\ 1 \end{bmatrix}$$

Start from $h^{(0)} = \begin{bmatrix} 0 \\ 0 \end{bmatrix}$ and compute forward pass step by step:

  • Step 1: $$z^{(1)} = W_{xh} x^{(1)} + W_{hh} h^{(0)} \quad h^{(1)} = \tanh(z^{(1)})$$
  • Step 2: $$z^{(2)} = W_{xh} x^{(2)} + W_{hh} h^{(1)} \quad h^{(2)} = \tanh(z^{(2)})$$
  • Step 3: $$z^{(3)} = W_{xh} x^{(3)} + W_{hh} h^{(2)} \quad h^{(3)} = \tanh(z^{(3)})$$

Each time, compute: $$y^{(t)} = W_{hy} h^{(t)}$$

Then use mean squared error or cross entropy loss to compute $\mathcal{L}^{(t)}$ for each time step.

Then for backpropagation:

  • Compute $\frac{\partial \mathcal{L}^{(3)}}{\partial h^{(3)}}$ from output layer.
  • Use recurrence to compute $\delta^{(2)}$ and then $\delta^{(1)}$.
  • Use all $\delta^{(t)}$ to compute gradients for each parameter.

This gives you full BPTT on a three step RNN.

Why Eigenvectors Matter for RNNs

In a vanilla RNN, the hidden state is updated recurrently as: $$h^{(t)} = W_{hh} h^{(t-1)}$$

When this happens repeatedly, we are effectively applying the matrix $W_{hh}$ many times: $$h^{(t)} = W_{hh}^t h^{(0)}$$

This means the long term behavior of the RNN is governed by how matrix powers $W_{hh}^t$ behave. And this is exactly what eigenvalues and eigenvectors tell us.

When analyzing the recurrent weight matrix $W_{hh}$, eigenvectors tell you which directions in the hidden state space remain consistent as the matrix is applied repeatedly.

  • If a hidden state vector aligns with an eigenvector of $W_{hh}$, then across time steps: $$W_{hh}^t v = \lambda^t v$$ That direction simply gets scaled, not rotated or altered, making it easier to predict how it evolves over time.

Predicting Behavior Using Eigenvectors

  • Stable directions: If an eigenvalue $\lambda \approx 1$, the state along its eigenvector remains steady.
  • Diverging directions: If $|\lambda| > 1$, activity in that direction grows exponentially, risking exploding gradients.
  • Fading directions: If $|\lambda| < 1$, memory in that direction fades, leading to vanishing gradients.

So yes, by analyzing the eigenvalues and eigenvectors, you can:

  • Understand the long term behavior of your RNN.
  • Choose smart weight initializations.
  • Know which directions dominate the dynamics and fine tune or regularize accordingly.

Problems with RNNs

Vanishing Gradients

  • During backpropagation through time (BPTT), the gradients involve repeated matrix multiplications: $$\delta^{(t)} = W_{hh}^T \delta^{(t+1)} \circ (1 - (h^{(t)})^2)$$
  • If the spectral radius of $W_{hh}$ (meaning $\max |\lambda|$) is less than 1, then: $$\delta^{(t)} \to 0 \text{ exponentially as } t \to 0$$
  • Consequence: Early layers receive almost no gradient, meaning the model cannot learn long term dependencies.

Exploding Gradients

This is related to the same effect of the $W_{hh}$ matrix being involved in so many matrix multiplications that at some point the gradient value can grow extremely large (when $\lambda_{\max} > 1$).

Short Term Memory Bias

  • Even when gradients do not vanish or explode, RNNs tend to bias heavily toward recent inputs.
  • They forget older information because: $$h^{(t)} = \tanh(W_{xh} x^{(t)} + W_{hh} h^{(t-1)} + b)$$ keeps overwriting $h^{(t-1)}$ at every time step.

No Memory Control

  • RNNs have no mechanism to:
    • Decide what to forget
    • Choose what to remember
    • Select what to expose
  • They always update the hidden state regardless of input, which leads to unstructured memory and information dilution.

Optimizer Used

Adam Optimizer

It handles vanishing and exploding gradients well since RNNs involve multiplying many Jacobians.

  • It has adaptive learning rates.
  • Momentum stabilizes gradients in BPTT.
  • Variance normalization prevents updates from becoming too small or too large.
