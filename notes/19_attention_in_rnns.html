Problem with RNNS:

The final encoder state that is passed to the decoder in seq2seq modules is just one token vector which is supposed to carry how important each token is in the sequence which is boderline impossible A better solution would be to pass all of the encoder hidden states to the decoder state and let it choose which parts are important for itself by assigning each part a score

So lets take the example of a simple sentence: This is an apple With attention the decoder learns to look at the object 'apple' being referred to as 'this' because let's say we're generating the word 'apple' in the decoder state then the word 'this' will be given more importance because we're referring to the apple

In RNNs, attention is only applied to Decoder as they use the encoder's hidden states while generating the output.

How it's done:

At each time step of the encoder we have hidden states: ,h2,h3....hT Decoder hidden state at time t= $s_{t}$

We give each hidden state a score based on how important/relevant it is to the decoder time state s $$e_{ti}=score(s_{t},h_{t})$$ In additive attention(Bahdanau attention): $$e_{ti}=v_{a}^T\tanh(W_{1}s_{t}+W_{2}h_{i})$$ where W1 and W2 are weight matrices (learned) and v is a weight vector (learned)

We use a weight vector instead of a matrix because the attention score is suppoesd to be one single scalar so we use it to get the score in 1d

In multiplicative attention: we use

a dot product: $$score(s_{t},h_{i})=s_{t}^Th_{i}$$ or a general form: $$score(s_{t},h_{i}=s_{t}^TW_{a}h_{i})$$ where W is a learnable weight matrix

Now that we have attention scores we convert to attention weights using softmax so that all of the scores are normalized/regularized $$\alpha_{ti}=\frac{\exp(e_{ti})}{\sum_{i}\exp(e_{ti})}$$ This gives us attention weights to every hidden state in the range of (0,1)

We use these attention weights and build a context vector

$$e_{t}=\sum_{i=1}^{T}\alpha_{ti}.h_{i}$$So we multiply each hidden state with attention weights to scale them by how important they are This context vector includes all of the hidden states and how important they are with respect to the current decoder state

Now this context vector is passed along with the decoder state to generate the next output $$s_{t}=\tanh(W_{c}[c_{t}];s_{t})$$ ; means we concatenate both the vectors together to give the final output

Bidirectional RNNS with Attention:

Encoder: Bidirectional

Each input token xi produces two hidden states:

  1. Forward RNN
  2. Backward RNN

we concatenate them to produce a token ht

so the encoder hidden state ht now contains information from both paste and future around token xi

Decoder: Unidirectional RNN with attention

The decoder works as usual from left to right and at each decoder time step we do the same things as in normal attention

  1. Calculate a score for the combined hiddden state and the current decoder state
  2. Find attention weights
  3. Make a context vector with the weighted sum of bidirectional encoder hidden states
  4. Use the context vector and generate output

Note:

In RNNS with attention we use both the final hidden encoder state as input for the decoder state and also the hidden states at every time step in attention to generate the next output So this means we use hidden states in:

  1. Start of decoder as final hidden stae
  2. Attention So in backprop, there are two routes that lead to the gradients of these states But autograd engines like the one in pytorch are so smart that due to them using computational graphs and stuff the total gradient of each hidden state is the sum of gradients through both the routes
$$\frac{\partial \mathcal{L}}{\partial h_3} = \frac{\partial \mathcal{L}}{\partial h_3}^{(1)} + \frac{\partial \mathcal{L}}{\partial h_3}^{(2)}$$
